1. 커밋 메시지 구조
type: subject (제목)
body (본문, 생략 가능)
footer (키워드, #이슈번호, 생략 가능)
#우리는 footer 사용X

2. type 7가지
1) feat: 새로운 기능
2) fix: 버그 해결, 기능 수정
3) style: html, css 등 디자인 요소 변경
4) refactor: 코드 리팩토링
5) test: 테스트 코드 추가 혹은 테스트 코드 리팩토링
# 우리는 5번까지만 사용

6) docs: 문서, 주석 변경
7) chore: 패키지 매니저 구성, 빌드 작업 업데이트

3. 규칙
1) 제목은 간략하게 작성
2) 본문(body)은 '어떻게'보다 '무엇을', '왜'에 초점을 맞춤
3) 커밋은 기능 단위로 함

4. 예시
1) test: 레시피 등록 기능 테스트
    파일첨부 기능 일부 완료

2) test: 레시피 등록 기능 테스트
    파일첨부 기능 구현 완료

2) feat: 레시피 등록 기능 추가

3) fix: 레시피 등록 기능 수정
   파일첨부 시 파일명이 깨져서 로직 변경
